Persistence Explorer plugin for NetBeans is a convenient tool that mimics the Databases modules in NetBeans, providing support to JPA 2.0 with the following features:

  • Manage the registry of persistence providers and persistence units in the Services explorer view of NetBeans.
  • Browse the metamodel of a registered persistence unit.
  • Execute queries from within a simple JPQL editor against a registered persistence unit, and view the results.

By now, it's just a experimental version with many limitations and known-issues, but it's still ideal for beginners of JPA 2.0 (like me:) to learn the API, run JPQL queries and then examine the results without the need to deploy your applications. Usage

The module adds a node named "Persistence" to the Services explorer view of the NetBeans. There are two child nodes, named "Persistence Providers" and "Persistence Units". The persistence providers node is a registry that allows user to register JPA 2.0 persistence provider implementations, and the persistence units node allows to register user persistence unit configuration files. Besides, the module depends on the Databases module's JDBC driver registry to "open" the registered user persistence units.

To work with the module, following below steps:

Prepare Your Persistence Unit

This module works ONLY with persistence unit configuration with the RESOURCE_LOCALE tranaction type. Package your entity classes and other required classes into jars together with a persistence.xml configuration file. Make sure they meet JPA 2.0 requirement for Java SE environment. For your convenience, here's an example of persistence.xml. 

  <persistence-unit name="HrEclipselinkResourceLocalPu" transaction-type="RESOURCE_LOCAL">
    <provider>org.eclipse.persistence.jpa.PersistenceProvider</provider>
    <class>org.javaplus.example.hr.Department</class>
    <class>org.javaplus.example.hr.Employee</class>
    <class>org.javaplus.example.hr.Project</class>
    <properties>
      <property name="eclipselink.logging.level" value="FINE"/>
      <property name="eclipselink.ddl-generation" value="create-tables"/>
      <property name="javax.persistence.jdbc.url" value="jdbc:mysql://localhost:3306/hr"/>
      <property name="javax.persistence.jdbc.password" value="hradm"/>
      <property name="javax.persistence.jdbc.driver" value="com.mysql.jdbc.Driver"/>
      <property name="javax.persistence.jdbc.user" value="hradm"/>
    </properties>
  </persistence-unit>

Register Your Persistence Unit

Once packaging is done, register your persistence unit to the explorer, by right-click the "Persistence Units", choose "New Persistence Unit", and fill up the form. Add all jars of your persistence unit to the list. The configuration files in all jars will be resolved automatically. Choose the configuration file and the unit name. Fill up other optional fields if needed. Make sure all files needed by your unit are registered, and the unit is valid.

Register Persistence Provider

The persistence provider implementaion must be registered in the explorer in order to open and work with your persistence units.

Open the provider registration form by right-click the persistence providers node, choose "New Peristence Provider", fill up the form by adding all required jars and naming the provider with a simple short name. Check your provider's documentation, make sure all needed jars are added.

Register JDBC Driver if Needed

Make sure the JDBC driver needed by your unit is registered in the Databases explorer of the NetBeans. The persistence explorer will try to find required JDBC driver from there.

Work with Your Persistence Unit

Before working with your persistence units, examine carefully the above steps you have done, make sure everything is ready. The module DOES NO validation at all by now, if anything goes wrong, it will just simply throw an ugly exception. You can figure out the problem by examining the stack trace. From the lower right corner of your IDE, you can find the exception indicator. Sorry for the inconvenience, I will improve this in the coming stable release.

Now you can open your persistence unit by right-click on it, choose "Open", if everything is ok, congratulations, you can browse the metamodel of your unit. Currently, the browsing of the metamodel is just experimental.

You can execute simple JPQL queries by choosing "Execute Command" from the persistence unit's context menu. That will take you to a simple plain JPQL editor. Tune your query, click "Execute Query", the result will show in the lower table view.
